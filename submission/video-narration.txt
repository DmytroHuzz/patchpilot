Dependency alerts are easy. Proving impact, and shipping a safe fix, is not. PatchPilot turns one real alert into a controlled, evidence-backed patch.

First, O. S. V. Scanner detects a high-severity advisory in J. S. O. N. five, version one point zero point one. The advisory and fix version are scanner facts—not model guesses.

Next, PatchPilot traces the dependency to the exact import and parse call. GPT five point six interprets only that bounded evidence, with citations, confidence, unknowns, and clear next checks.

Before anything changes, PatchPilot presents the complete four-file plan: the version update, source repair, regression test, verification commands, risks, and rollback. A human approves that exact plan.

Only then does PatchPilot create an isolated Git worktree. Every write stays off the clean source checkout, and exact diff checkpoints prevent the patch from drifting beyond the approval.

The approved update changes the manifest and lockfile. The model proposes one constrained function repair, and a benign allowlist test proves the intended behavior without introducing an exploit sample.

Now the deterministic checks take over: clean install, existing tests, targeted test, full suite, build, and a lockfile-scoped rescan. The selected advisory is gone. PatchPilot claims only what the evidence proves.

The final report preserves the original finding, repository citations, model interpretation, human approval, exact diffs, command results, clean rescan, remaining uncertainty, and next checks—in Markdown and JSON.

Finally, PatchPilot commits exactly four approved files and prepares accurate pull-request copy. Publishing stays locked behind a separate approval. One alert enters; a reviewable, verified fix comes out.
